# 营销游戏竞品分析 Skill

## 核心能力

该 Skill 面向“关键词驱动的竞品调研”，支持完整闭环：
- 自动打开小红书搜索页并按关键词检索
- 自动下滑浏览帖子流，按用户要求抽样
- 点开帖子详情并下滑评论区
- 提取正文、互动数据、评论观点并保存截图证据
- 先输出“图文对应确认稿”给用户确认
- 用户确认后输出 HTML/PNG 格式报告，且结论逐条对应截图证据

## 适用场景

- 竞品内容策略分析
- 用户口碑与评论情绪分析
- 热门话题与卖点提炼
- 活动期（如 618）平台内容对比

## 输入约定

优先接收以下输入（缺失时主动补默认值并告知用户）：
- 关键词：如“618 护肤 礼盒”
- 竞品范围：品牌 A / 品牌 B / 不限
- 样本量：默认 10 篇（可改 5/20）
- 评论深度：默认每篇前 20 条可见评论
- 输出目标：洞察摘要 / 竞品对比 / 行动建议

## 执行流程

### 步骤 1：建立任务目录

```bash
mkdir -p /tmp/xiaohongshu-analysis/{screenshots,data,report}
```

建议按时间戳创建子目录，避免任务互相覆盖。

### 步骤 2：自动打开小红书并搜索关键词

1. 将关键词 URL 编码后拼接搜索链接：
   `https://www.xiaohongshu.com/search_result?keyword=<encoded_keyword>`
2. 优先使用可自动化浏览方式打开页面并执行后续操作。
3. 如遇登录墙，提示用户在弹出的浏览器窗口完成一次登录后继续。

说明：
- 小红书详情页/评论区常需要登录后可见。
- 登录完成后在当前会话内继续抓取，减少重复登录。

### 步骤 3：自动下滑列表并采样帖子

按用户要求执行滚动与采样：
- 滚动搜索结果页，采集候选帖子卡片
- 过滤广告/无关帖
- 按热度与相关度保留样本

每条候选至少记录：
- 帖子链接
- 帖子标题（或首句）
- 作者名
- 点赞/收藏/评论（页面可见值）
- 列表页截图路径

### 步骤 4：逐条进入帖子详情并抓评论

对每条入选帖子执行：
1. 打开帖子详情页
2. 截取帖子头图/正文关键段
3. 先定位评论区容器（用户可见红框区域），必须在该容器内单独滚动，不得仅滚动整页
4. 在评论区容器内连续滚动浏览（默认至少 3 次，每次间隔 0.8-1.2 秒），确保出现新的评论内容
5. 提取热门评论与情绪倾向
6. 保存“结论证据截图”（截图时需包含评论区标题和多条评论正文）

评论区滚动执行标准（强制）：
- 每次采集都必须执行“评论容器内滚动”，不允许跳过。
- 若容器不可滚动，才允许回退到整页滚动，并在结果中标记“容器滚动失败（已回退）”。
- 若滚动后评论未变化，需增加滚动次数或等待后重试一次。

截图命名规范（必须执行）：
- `post_<n>_detail.jpg`：帖子详情主内容截图
- `post_<n>_comments.jpg`：评论区截图

### 步骤 5：产出“图文对应确认稿”（先不给最终报告）

先向用户输出确认稿，格式示例：

```markdown
# 小红书样本确认稿

## 样本 1
- 链接：...
- 摘要：...
- 初步判断：...
- 证据截图：
  - 详情页：file:///tmp/xiaohongshu-analysis/screenshots/post_1_detail.jpg
  - 评论区：file:///tmp/xiaohongshu-analysis/screenshots/post_1_comments.jpg

## 样本 2
...
```

确认规则：
- 用户明确回复“确认/可以生成报告/按此输出”后，才进入最终报告阶段。
- 若用户要求替换样本或补抓评论，先回到步骤 3/4。

### 步骤 6：生成最终竞品分析报告（HTML + PNG）

用户确认后，生成：
- HTML 报告：`/tmp/xiaohongshu-analysis/report/competitor_report.html`
- PNG 报告：`/tmp/xiaohongshu-analysis/report/competitor_report.png`

PNG 生成方式：
- 使用浏览器打开 HTML 后截图导出整页 PNG。

## 报告结构

### 1. 任务概览
- 关键词、时间范围、样本量、筛选规则

### 2. 样本总览
- 样本清单表（链接、作者、互动数据）

### 3. 竞品对比
- 内容主题分布
- 卖点表达差异
- 用户情绪与高频关注点
- 评论区典型观点

### 4. 关键结论（必须包含图文证据）
- 3-5 条结论，必须可追溯
- 结论可以由多个帖子共同归纳得出，不要求“一条帖子对应一条结论”
- 每条结论下方直接展示相关证据截图缩略图，不再单独输出“结论-证据映射表”章节
- 最终 HTML/PNG 报告中必须直接展示证据截图，不得只展示图片地址

推荐 HTML 结构示例：

```html
<div class="insight">
  <b>C1：用户核心诉求是设置和触发</b>
  <p>结论说明...</p>
  <div class="evidence-card">
    <img src="file:///tmp/xiaohongshu-analysis/screenshots/post_2_comments.jpg" alt="样本2评论区证据">
    <p>样本 2 评论区：收集讨论</p>
  </div>
</div>
```

### 5. 可执行建议
- 面向运营/内容/投放的落地建议

## 输出质量标准

- 图文对应按内容相关性组织：每条核心结论至少 1 张截图证据，可对应多条帖子、多张截图；不强制一条帖子一条结论
- 最终 HTML/PNG 报告必须直接嵌入展示证据截图（`<img>` 缩略图/卡片），不得只展示 `file://` 图片地址；图片地址可作为辅助链接保留
- 不编造数据：只使用页面可见信息
- 评论分析要标注样本范围（例如“每帖前 20 条可见评论”）
- 评论采集必须基于“评论区容器内滚动后”的可见内容（非整页首屏）
- 先确认后出报告：不得跳过用户确认环节
- 报告可复核：必须保留截图与原始链接

## 异常处理

### 场景 1：无法登录小红书
- 提示用户在可见浏览器窗口手动登录
- 登录后从当前步骤继续，不重置任务目录

### 场景 2：帖子打不开或被删除
- 标记为无效样本并替补新样本
- 在最终报告中注明替补规则

### 场景 3：评论区加载不全
- 增加滚动次数与等待时长后重试
- 若仍不完整，在报告中标注“评论可见范围限制”

### 场景 4：关键词结果偏离需求
- 调整关键词组合（品牌词+品类词+场景词）
- 向用户展示新采样预览并再次确认
