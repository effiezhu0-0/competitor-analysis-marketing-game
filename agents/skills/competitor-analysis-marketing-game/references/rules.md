# Rules

## 核心规则

1. 先确认样本，再生成最终报告。
2. 评论采集必须进入帖子详情页，并浏览评论区。
3. 评论区滚动必须优先滚动评论区容器，不得只滚动整页。
4. 证据截图只需要详情页和评论区，不需要在确认稿中展示列表页。
5. 最终 HTML/PNG 报告必须直接嵌入展示证据截图，不能只展示 `file://` 图片地址。
6. 结论可以由多个帖子共同归纳，不要求一条帖子对应一条结论。
7. 结论与截图按内容相关性对应，一个结论可以对应多张相关截图。
8. 不编造数据，只使用页面可见信息和已采集证据。
9. 评论分析需要说明样本范围，例如“基于页面可见评论”。
10. 若小红书要求登录，必须提示用户在浏览器中完成登录后继续。

## 评论区滚动规则

- 默认至少在评论区容器内滚动 3 次。
- 每次滚动后等待 0.8-1.2 秒，观察是否出现新评论内容。
- 如果容器不可滚动，可回退整页滚动，但需要标记“容器滚动失败，已回退”。
- 评论截图应包含评论区标题或评论数量，以及多条评论正文。

## 截图规则

确认稿中每条样本只展示：

- 详情页截图
- 评论区截图

最终报告中：

- 关键结论下方应直接展示相关证据截图缩略图。
- 不再单独输出冗余的“结论-证据映射表”章节。
- 截图下方应有简短说明，例如“样本 4 评论区：设置咨询”。

## 报告结构规则

最终报告建议包含：

1. 任务概览
2. 样本总览
3. 竞品对比
4. 关键结论（含图文证据）
5. 样本图文证据
6. 可执行建议

## 禁止事项

- 禁止跳过样本确认直接输出最终报告，除非用户明确要求基于已有确认样本继续。
- 禁止用未查看评论区的样本支撑评论结论。
- 禁止把登录弹窗截图当作有效证据图。
- 禁止只展示图片路径而不展示图片。
- 禁止把临时目录 `/tmp` 作为唯一交付位置。
- 禁止捏造评论、点赞、收藏、评论数或用户观点。

## 数据质量提示

- 小红书页面结构可能变化，评论文本提取可能包含拼接噪声。
- 当文本提取不稳定时，应以截图证据为主，文字总结为辅。
- 如果截图被弹窗遮挡，需要重新登录或重新截图。
