# Workflow

## 阶段 1：理解需求

确认用户的目标：

- 搜索关键词
- 样本数量
- 是否需要进入详情页
- 是否需要查看评论区
- 输出形式：确认稿 / 总结 / HTML / PNG
- 是否已有样本和截图，是否禁止重新搜索

如果用户明确说“不要重新搜索”或“基于已有信息”，必须复用已有信息。

## 阶段 2：建立稳定工作目录

优先使用当前项目下的稳定目录，例如：

```text
<project>/xiaohongshu-analysis/<task-name>/
├── screenshots/
├── data/
└── report/
```

不建议只使用 `/tmp`，因为系统可能清理临时文件。

## 阶段 3：搜索与采样

当用户允许重新采集时：

1. 打开小红书搜索页。
2. 输入或拼接关键词搜索 URL。
3. 滚动搜索结果页获取候选帖子。
4. 过滤广告、无关内容、重复链接。
5. 记录入选样本链接、标题、作者和互动信息。

## 阶段 4：详情页和评论区采集

对每条样本：

1. 打开帖子详情页。
2. 截取详情页截图。
3. 定位评论区容器。
4. 在评论区容器内单独滚动浏览。
5. 截取评论区截图。
6. 提取可见评论文本和初步观点。

## 阶段 5：输出样本确认稿

确认稿包含：

- 链接
- 摘要
- 初步判断
- 详情页截图
- 评论区截图

确认稿不展示列表页截图。

## 阶段 6：用户确认

用户明确回复“确认”“可以生成报告”“按此输出”等，再进入最终报告阶段。

如果用户要求替换样本或补抓评论，则回到采集阶段。

## 阶段 7：生成最终报告

最终报告应包含：

- 任务概览
- 样本总览
- 竞品对比
- 关键结论
- 样本图文证据
- 可执行建议

关键结论下方直接展示证据截图缩略图。一个结论可以对应多个相关样本和多张截图。

## 阶段 8：导出和交付

输出：

- HTML 报告
- PNG 报告
- 稳定本地路径

报告文件应放在稳定目录中，并用可点击 `file://` 链接返回给用户。

## 异常处理

### 登录弹窗

提示用户登录后继续。登录弹窗截图不能作为有效证据。

### 评论区无法滚动

回退整页滚动，并在结果中说明。

### 截图丢失

如果截图在临时目录被清理，应说明原因，并建议使用稳定目录重新生成。

### 用户禁止重新搜索

只基于已有信息输出；如果缺少截图，需要明确说明不能补图，除非用户允许重新打开已有链接截图。
