# Overview

## 背景

营销游戏和互动活动通常依赖社媒平台进行种草、传播和用户反馈收集。小红书评论区中常出现用户对玩法、设置方式、触发机制、奖励机制和情绪体验的真实反馈，因此适合作为竞品分析的重要信息源。

该 Skill 将“小红书搜索、帖子抽样、详情查看、评论区浏览、截图取证、确认稿、最终报告”沉淀为标准流程，帮助 Zero 在类似任务中稳定产出可复核的竞品分析结果。

## 适用范围

适用于以下对象：

- 营销游戏
- 互动活动
- 小组件 / 桌面组件
- 品牌活动玩法
- 社媒内容传播
- 用户评论洞察

## 核心概念

### 样本确认稿

最终报告前的中间产物。它用于让用户确认样本是否可用，通常包含链接、摘要、初步判断、详情页截图和评论区截图。

### 评论区容器滚动

评论区通常是页面中的独立滚动区域。采集评论时必须优先滚动评论区容器，而不是只滚动整页。只有评论容器不可滚动时，才允许回退整页滚动，并需要标注。

### 图文对应

报告中的结论必须有截图证据支撑。一个结论可以由多个帖子、多张详情页或评论区截图共同支撑，不要求一条帖子对应一条结论。

### 可见评论范围

评论采集只代表页面当前可见和滚动后加载出的评论，不等于平台全部评论。报告中需要说明样本范围和限制。

## 典型分析维度

- 内容主题
- 卖点表达
- 互动机制
- 评论区高频问题
- 用户情绪
- 转化机会
- 可复用内容模板
