# competitor-analysis-marketing-game

## 基础信息

- 提交人：effiezhu0-0
- 维护人：effiezhu0-0

- 当前状态：已应用 v1.4.0

## 用途

该 Skill 用于帮助 Zero 执行营销游戏、互动活动和社媒内容的竞品分析任务。它可以围绕关键词检索小红书内容，查看帖子详情和评论区，保存证据截图，并在用户确认样本后生成 HTML/PNG 格式的竞品分析报告。

## 适用场景

- 小红书竞品内容分析
- 营销游戏或互动活动的传播复盘
- 用户评论观点总结与情绪分析
- 评论区高频诉求和玩法机会提炼
- 活动玩法、内容结构、互动机制对比
- 需要图文证据支撑的竞品分析报告

## 输入

用户通常需要提供：

- 关键词，例如“蚂蚁庄园桌面组件”
- 样本数量，例如前 3 条、前 6 条、前 10 条
- 是否需要查看详情页与评论区
- 输出形式，例如确认稿、HTML 报告、PNG 报告
- 可选维度，例如内容吸引点、互动机制、用户需求、转化机会

## 输出

Skill 通常产出：

- 样本确认稿
- 详情页截图
- 评论区截图
- 评论区要点总结
- 竞品分析结论
- HTML 报告
- PNG 报告

## 边界和风险

- 小红书页面可能要求登录，需要用户先完成登录。
- 评论数据只代表页面可见范围，不等于全量评论。
- 不能编造互动量、评论内容或用户观点。
- 最终报告必须展示证据截图，不能只给图片地址。
- 截图应保存到稳定目录，避免使用容易被清理的 `/tmp` 作为唯一保存位置。

## 可选补充

- 典型案例：围绕“蚂蚁庄园桌面组件”分析小红书帖子内容、评论区需求和互动机制。
- 主要收益：将重复的搜索、截图、评论总结和报告生成流程标准化，降低竞品分析成本。
