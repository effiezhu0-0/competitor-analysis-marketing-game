# 营销游戏竞品分析 Skill

## 基础信息

- 所属业务域：主站营销互动
- 当前状态：可在Zero中使用，v1.0

## 用途

围绕小红书等社媒平台的营销游戏、互动活动和内容传播案例，自动执行关键词检索、帖子详情浏览、评论区容器滚动、证据截图保存和评论观点归纳，帮助 Zero 产出可确认、可追溯、图文对应的竞品分析报告。

该 Skill 适合用于活动玩法调研、评论区洞察、社媒内容复盘和互动机制机会提炼。执行时会先输出样本确认稿，待用户确认后再生成 HTML / PNG 格式的最终竞品分析报告。

## 适用场景

- 需要分析小红书上某个营销游戏、互动活动或社媒话题的传播表现
- 需要查看多条帖子详情和评论区，提炼用户真实反馈、情绪和高频诉求
- 需要对比竞品内容结构、互动机制、用户吸引点和转化机会
- 需要把零散帖子、评论和截图整理成结构化竞品分析报告
- 需要图文证据支撑结论，且最终 HTML / PNG 报告中直接展示证据截图
- 用户提到「小红书竞品分析」「营销游戏分析」「互动活动复盘」「评论区洞察」「帖子评论分析」等需求时

## 输入

用户通常需要提供：

| 输入项 | 是否必填 | 说明 |
|--------|----------|------|
| 搜索关键词 / 竞品方向 | 必填 | 如「蚂蚁庄园桌面组件」「治愈游戏」「养成游戏」 |
| 样本数量 | 必填 | 如前 3 条、前 6 条、前 10 条；可分批采集 |
| 评论区浏览要求 | 必填 | 默认进入帖子详情页，并在评论区容器内单独滚动浏览评论 |
| 输出形式 | 必填 | 样本确认稿、HTML 报告、PNG 报告，或分阶段输出 |
| 分析维度 | 选填 | 如内容吸引点、互动机制、用户需求、情绪倾向、转化机会 |
| 平台登录状态 | 条件必填 | 如小红书要求登录，用户需在内置浏览器中手动完成登录 |

示例提问：

```text
搜索「蚂蚁庄园桌面组件」并点击前 6 条帖子查看评论，
注意单独滑动评论区浏览。先把收集到的信息总结给我，
我确认后再输出 HTML 和 PNG 格式的竞品分析报告。
```

## 输出

Skill 执行后会产出：

1. **样本确认稿**
   - 每条样本的标题、正文要点、互动信息和可见评论摘要
   - 详情页截图路径
   - 评论区截图路径
   - 评论区情绪、用户诉求和可用洞察

2. **竞品分析结论**（用户确认样本后）
   - 3-5 条关键结论
   - 结论可以由多个帖子共同归纳，不要求一条帖子对应一条结论
   - 每条结论下直接展示相关证据截图
   - 明确样本范围，例如“基于页面可见评论”

3. **HTML / PNG 报告**
   - HTML 报告直接内嵌详情页和评论区截图，不只展示图片地址
   - PNG 报告用于快速预览和分享
   - 报告包含样本说明、洞察总结、玩法机会和证据图片

最终回复应包含：样本确认结果、报告文件路径、截图目录路径，以及无法采集或需要用户登录时的明确说明。

## 边界和风险

**不能做：**

- 绕过平台登录、付费墙、私有内容或反爬限制
- 代替用户输入账号密码
- 编造互动量、评论内容、用户观点或未采集到的结论
- 只滚动整页却声称已浏览评论区；必须优先在评论区容器内单独滚动
- 在用户未确认样本前直接生成最终竞品分析报告
- 在最终 HTML / PNG 报告中只放图片地址而不展示证据截图

**必须人工审核：**

- 小红书登录状态；如出现登录弹窗，需要用户手动完成登录
- 样本是否符合用户要分析的关键词、竞品方向和数量要求
- 评论区截图是否真正包含可见评论，而不是登录弹窗或空白区域
- 最终结论是否符合业务语境，是否可以用于对外汇报或决策

**其他风险：**

- 评论数据只代表页面可见范围，不等于全量评论
- 小红书页面结构可能变化，评论区容器定位和滚动方式需要按实际页面调整
- 平台可能限制内容访问或截图，导致部分帖子无法完整采集
- 截图和报告应保存到稳定目录，避免只保存在容易被清理的 `/tmp` 目录
